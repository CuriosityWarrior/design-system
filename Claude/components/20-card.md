# 컴포넌트: 카드 (Card)

## 개요
관련 콘텐츠와 액션을 묶는 컨테이너 서피스.

---

## 디자인 토큰

| 토큰 | 값 |
|---|---|
| 배경 | `color/surface/1` |
| 보더 | 1px solid `color/border/default` |
| 보더 반경 | `radius/card` = 12px |
| 그림자 | `shadow/card` |
| 헤더 패딩 | 16px 20px |
| 본문 패딩 | 20px |
| 푸터 패딩 | 14px 20px |
| 헤더 보더 | 1px solid `color/border/subtle` |
| 푸터 보더 | 1px solid `color/border/subtle` |
| 푸터 배경 | `color/surface/2` |

---

> 📐 사이즈 기본 원칙은 [파운데이션: 사이즈](../foundations/07-size.md) 참조. 보더는 `size/border/thin`(1px) 적용.

## 구조
- **헤더** (선택): 제목 + 부제목 + 우측 액션 (버튼/아이콘)
- **본문**: 주요 콘텐츠 영역
- **푸터** (선택): 액션 버튼 (`02 — Button` 페이지의 Button 컴포넌트 인스턴스), 우측 정렬

---

## 변형

### 기본 카드
- 정적, 비인터랙티브
- 전체 헤더/본문/푸터 구조

### 클릭 가능 카드
- 커서: pointer
- 호버: 그림자 깊어짐, translateY(-1px)
- 전환: `motion/hover` = 100ms ease-out
- 탐색 카드, 항목 카드에 사용

---

## 사이즈 동작

| 속성 | 값 |
|---|---|
| layoutSizingHorizontal | `FILL` 또는 `HUG` (배치 컨텍스트) |
| layoutSizingVertical | `HUG` |

> Height는 내부 콘텐츠 + 패딩 기준으로 자동 조정된다. 임의 px 값으로 Fixed 지정 금지. (그리드에서 카드 높이를 맞춰야 할 경우 부모 그리드에서 `FILL`로 동기화)

---

## 아이콘 사용 규칙

> 컴포넌트 내 아이콘은 반드시 `01 — Icons` 페이지의 아이콘 컴포넌트 인스턴스를 사용한다.
> 텍스트 특수 문자(✓, ✕, →, ⋯ 등), 이모지, 직접 그린 벡터 도형으로 아이콘을 대체하는 것을 금지한다.
> 필요한 아이콘이 없는 경우, 먼저 `01 — Icons` 페이지에 추가한 후 인스턴스를 참조한다.

## Figma Make 프롬프트

```
다음 스펙으로 카드(Card) 컴포넌트를 만들어줘:

배경: 흰색 (라이트) / 다크 서피스 (다크)
보더: 1px solid 연한 회색
보더 반경: 12px
미세한 드롭 섀도우

선택적 섹션을 포함한 구조:
- 헤더: 제목 (16px SemiBold) + 선택적 부제목 (13px 회색) + 선택적 우측 액션
  - 본문과 구분하는 하단 보더
- 본문: 주요 콘텐츠, 패딩 20px
- 푸터: 우측 정렬 Button 컴포넌트 인스턴스들, 상단 보더, 약간 어두운 배경

변형:
- 기본 (정적)
- 클릭 가능 (호버: 그림자 깊어짐 + 약간 위로 이동)

네이밍: Card / Default, Card / Clickable
```
